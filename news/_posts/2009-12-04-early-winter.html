Release v1.4.1

Early Winter 2009

This feature release adds features directly from git-colaâ€˜s github issue tracker. On the developer front, further work was done towards modularizing the code base.

Usability, bells and whistles

  • Dragging and dropping patches invokes git-am (closes #3)
  • A dialog to allow opening or cloning a repository is presented when git-cola is launched outside of a git repository. (closes #22)
  • Warn when push is used to create new branches (closes #35)
  • Optimized startup time by removing several calls to git.

Portability

  • git-cola is once again compatible with PyQt 4.3.x (closes #34)

Developer

  • cola.gitcmds was added to factor out git command-line utilities
  • cola.gitcfg was added for interacting with git-config
  • cola.models.browser was added to factor out repobrowser data
  • Added more tests
