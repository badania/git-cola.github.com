News

Release v1.4.0

Halloween 2009

The focus for this release has been the on a redesign of the main git-cola interface and some new minor features.

Changes since v1.3.9

  • Flexible, configurable GUI
  • Commit message editor displays current row/column and warns when lines get too long.
  • The "classic" view displays paths that have been changed upstream
  • New interface for creating tags
  • Lots of miscellaneous bug fixes
  • 'git cola --classic' launches the classic view in standalone mode
  • The GUI inherits the main system-configured UI font
  • Built-in/fallback SSH_ASKPASS prompts for SSH passwords on fetch/push/pull
  • More logging and user feedback for commands

Technical updates:

  • Removed all usage of Qt Designer .ui files
  • Simpler model/view architecture
  • Rewritten status view
  • Selection now lives in the model
  • Centralized GUI notifications to keep views in sync
  • Less coupling between views and model actions
  • Added thread-safety to the git command class
  • More documentation and tests
